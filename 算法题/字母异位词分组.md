给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

 

示例 1:

输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]

输出: [["bat"],["nat","tan"],["ate","eat","tea"]]

解释：

在 strs 中没有字符串可以通过重新排列来形成 "bat"。
字符串 "nat" 和 "tan" 是字母异位词，因为它们可以重新排列以形成彼此。
字符串 "ate" ，"eat" 和 "tea" 是字母异位词，因为它们可以重新排列以形成彼此。


核心在于找到关键的 特征 ，可以发现通过排序就可以把这个关键的特征给找出来。了解大概以后，可以联想到用map集合来实现，key就是排序后的特征，value就是原本的那个。

有个注意事项就是最后收集，map.values()返回所有 Value 的 Collection 集合；

数组 length vs 集合 size ()
数组（如String[] strs）是固定长度的，用strs.length获取长度；
集合（如List<String>）是动态长度的，用list.size()获取元素个数；
👉 这是 Java 中数组和集合的核心区别，记准即可。

List 是接口，不能直接 new，必须 new 它的实现类（ArrayList 是最常用的）；

new list<String> l = map.get(c);这个语法正确吗？
这个语法完全不正确  map.get(c)是引用  。多余的 new 关键字（逻辑 + 语法双错）
new 关键字的作用是创建新的对象（比如 new ArrayList<>()），但 map.get(c) 是 “获取 map 中已存在的 List 对象”，不是创建新对象 —— 你不需要、也不能在获取已有对象时加 new。
比如：map 中已经存了 ["eat"] 这个 List，map.get(c) 是拿到这个现成的 List，而不是新建一个；
加了 new 后，编译器会认为你想 “创建新对象”，但后面又跟了 map.get(c)（取值），语法完全冲突。
