给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

 

示例 1：

输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

本题的最重要的要求就是时间复杂度有要求，空间换时间

第一步：先想 “暴力思路”（理解问题本质）
最长连续序列的核心特征是：序列中的每个数 x，都能找到 x+1、x+2... 直到找不到为止。比如对于数字 1，找 1+1=2，2+1=3，3+1=4，4+1=5（找不到），所以长度是 4。
基于这个特征，暴力思路的步骤可以拆成：
遍历数组中的每一个数 num；
以 num 为起点，依次检查 num+1、num+2... 是否存在于数组中；
记录每次检查得到的连续长度，最终取最大值。
👉 你先思考：如何快速判断一个数 “是否存在于数组中”？（提示：如果直接遍历数组查，时间复杂度是 O (n)，整体会到 O (n²)；有没有数据结构能把 “查询是否存在” 的时间降到 O (1)？）
第二步：优化 “查询效率”（核心突破口）
这道题的关键是用空间换时间，把数组中的数存入一个 “支持快速查询” 的集合中：
Java 中可以用 HashSet（和 HashMap 类似，但只有 Key，没有 Value），因为HashSet.contains() 方法的时间复杂度是 O (1)；
先把数组所有元素存入 HashSet，目的是：后续检查num+1是否存在时，能瞬间判断。
👉 你先思考第一步具体操作：如何把数组 nums 存入 HashSet？（回忆之前 HashMap 的创建 / 添加操作，HashSet 的用法几乎一致，比如 HashSet<Integer> set = new HashSet<>();，添加用 set.add(num);）
第三步：避免重复计算（关键优化点）
暴力思路有个致命问题：会重复遍历。比如数组[1,2,3,4]，遍历到 1 时会查 1→2→3→4，遍历到 2 时又会查 2→3→4，完全没必要。核心优化点：只有当一个数 num 是 “连续序列的起点” 时，才开始统计长度。那么怎么判断 num 是起点？→ 如果 num-1 不存在于集合中，说明 num 是连续序列的第一个数（比如 1 的num-1=0不存在，所以 1 是起点；而 2 的num-1=1存在，所以 2 不是起点，不用统计）。
👉 你先梳理这个逻辑：遍历集合中的每个数 num，先判断 num-1 是否在集合中。如果不在，就以num为起点，统计连续长度；如果在，直接跳过。

特征	HashMap	HashSet
存储结构	存储 Key-Value 键值对	只存储 Key（Value 是固定的空对象，用户不可见）
核心用途	需建立 “键→值” 的映射（如字母异位词分组）	只需判断 “元素是否存在”（如最长连续序列查 num-1 是否存在）
常用方法	put()/get()/containsKey()/values()	add()/contains()/remove()
是否允许重复	Key 不可重复，Value 可重复	元素（即 Key）不可重复
是否允许 null	Key 可以有一个 null，Value 可多个 null	可以有一个 null 元素


特征	普通数组（如 int []）	HashSet（如 HashSet<Integer>）
查询元素是否存在	必须遍历数组，时间复杂度 O (n)	哈希表直接定位，时间复杂度 O (1)
是否自动去重	允许重复元素（比如数组 [1,1,2]）	自动去重（add 重复元素会被忽略）
存储顺序	按插入顺序保存，有序	无序（基于哈希值存储，和插入顺序无关）
访问方式	按索引访问（arr [0]），高效	无法按索引访问，只能遍历 / 判断存在性
扩容 / 修改	长度固定，扩容需手动新建数组	动态扩容，add/remove 自动处理
